One-Time-URL

This example shows how to use OTURL within an integration schema where an iframe-embedded RDP connection interacts with the parent webpage.

The On-Time-URL feature (OTURL) helps developers and integrators to create discardable, temporary connections using access profiles and User/Password security levels.

These are some situations in which the One-Time-URL might be useful:

  • Giving access to a desktop or application to external users without having to weaken the Security level to None.
  • Generating temporary access to a desktop.
  • Integrating <%=$LICPROD%> on a Single-Sign-On Scheme along with external applications.

To test this example, please fill the next textbox with a generated One-Time-URL address. If you don't have one, you can build it by using the One-Time-URL generator example.


The JavaScript SDK

The JavaScript SDK library allows you to integrate a fully functional remote desktop or remote application (including RemoteApp) inside other websites or web applications.

You have three ways to configure this feature: by using a pre-defined profile, defining all configurations items from JavaScript without a pre-defined profile, or mixing both systems, starting with a pre-defined profile but changing or adding some configuration items from JavaScript.
